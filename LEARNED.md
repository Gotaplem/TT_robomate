LEARNED — FastAPI

Изучено в процессе выполнения:
- Создание REST API эндпоинтов с FastAPI
- Использование Pydantic для валидации данных
- Тестирование API через fastapi.testclient
- Автоматическая документация через /docs

Как интегрировал в решение:
- Модули: app/main.py, app/routes.py
- Сервисы: API для приёма событий и получения статистики
- Использовано для запуска локального сервера и тестирования эндпоинтов

Плюсы / Минусы / Trade-offs

Плюсы:
- Лёгкий запуск и развёртывание
- Автоматическая документация /docs
- Поддержка асинхронных запросов
- Простой тестовый клиент

Минусы:
- Необходимость изучения асинхронности при масштабном приёме событий
- Ограничение производительности при очень больших потоках без дополнительных настроек

Когда использовать / когда избегать:

Использовать, если:
- Нужно быстро реализовать API для микросервисов
- Достаточно простой интеграции с Pydantic и SQLAlchemy

Избегать, если:
- Нужен высоконагруженный потоковый ingest с миллионами событий в секунду
- Требуется сложная асинхронная логика без опыта работы с asyncio
