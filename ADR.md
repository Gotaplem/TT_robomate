Решение: Выбор базы данных и фреймворка для Event Analytics Service

Контекст
Цель: собирать аналитические события от пользователей и предоставлять статистику (DAU, топ-события, retention). 
Нужен бэкенд с возможностью быстрого приёма событий и простого запуска сервиса.

Варианты

Вариант A: SQL (SQLite / PostgreSQL)
- Плюсы: простая интеграция, транзакции, понятный синтаксис
- Минусы: для больших потоков событий возможны ограничения по производительности

Вариант B: NoSQL (MongoDB)
- Плюсы: гибкая схема JSON, быстрый запись
- Минусы: сложнее строить аналитические запросы (агрегации)

Вариант C: Колонковая база (DuckDB / ClickHouse)
- Плюсы: быстрые аналитические запросы, удобно для batch
- Минусы: сложнее масштабирование для онлайн-ingest, требует дополнительной интеграции

Решение
Выбрана SQLite (файловая база данных) для простоты запуска и тестирования локально. Для аналитики используются SQL-запросы через SQLAlchemy. В будущем, для больших объёмов данных, можно заменить на ClickHouse или DuckDB для холодного слоя.
